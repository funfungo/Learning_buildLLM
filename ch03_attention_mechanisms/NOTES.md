> 本文件汇总了学习第 2、3 章过程中讨论清楚的核心问题，按主题归类。

## 一、Embedding 与词向量基础

### Q1. Token ID 之间能算相似度吗？
**不能。** ID 只是"门牌号"，本身无语义。
真正能算相似度的是**向量**——通过 `nn.Embedding` 查找表把 ID 映射成向量。

```
文本 → Token → ID → Embedding 表查找 → 向量 → 点积 → 相似度
```

### Q2. Embedding 表本身是"训练好的模型"吗？
- **刚 `nn.Embedding(...)` 创建时**：随机初始化，无语义
- **配合完整模型训练后**：表里向量自然学到语义结构
- **特例**：Word2Vec / GloVe 是单独训练 Embedding 的"古典模型"，可以独立使用
- **现代 LLM**：Embedding 只是"起点"，真正语义理解在后续 Transformer 层

第 2 章演示用的是**随机初始化**的表——机制对，但向量无意义。第 5 章训练后才"活过来"。

### Q3. Token ID 在不同模型间通用吗？
**不通用。** Tokenizer 和 Embedding 表必须**配对使用**：
- 同一个词 "hello" 在 GPT-2 / GPT-4 / BERT / LLaMA 里 ID 完全不同
- 同一个 ID 在不同模型的 Embedding 表里指向完全不同的向量
- 类比：Tokenizer = 图书馆编目体系，Embedding 表 = 书架位置

跨模型迁移时只能"先解码回文本，再用目标模型重新编码"。

### Q4. 词向量相似度的本质是什么？
**点积**：`a · b = |a||b|cos(θ)`

- 几何意义：点积越大 → 方向越接近 → 越相似
- 直觉理解：每一维代表一种"语义特征"，点积衡量两个词在所有维度上"评分一致程度"
- 同号项相加（贡献正分）→ 相似；异号项相加（贡献负分）→ 不相似

## 二、注意力机制核心

### Q5. 注意力机制的本质是什么？
**加权平均** —— 让每个 token 根据"相关度"融合其他 token 的信息：

```
输出 = Σ (相关度权重) × (内容)
```

### Q6. Q/K/V 三件套的作用？
类比搜索引擎：
- **Q (Query)**：我想找什么？
- **K (Key)**：我有什么标签可以被找到？
- **V (Value)**：我提供什么内容？

公式：`Attention(Q,K,V) = softmax(QK^T / √d_k) · V`

### Q7. 为什么 Q ≠ K（要分开两个矩阵）？
如果共用一个矩阵，"A 关注 B" 和 "B 关注 A" 程度相同（对称矩阵）。
分开 Q/K 让注意力变成**有方向性**的关系，更符合实际语言（如 "他打了球" 中 "球" 关注 "打" 强于反向）。

## 三、Scaling 与 Softmax

### Q8. 为什么要除以 √d_k？
- 点积方差随维度增长：`Var(q·k) = d_k`
- d_k 大时，点积数值大 → softmax 输出近似 one-hot → **梯度消失**
- 除以 √d_k 把方差归一化到 1，保持权重适度平滑、梯度健康

这就是 "**Scaled** Dot-Product Attention" 名字的由来。

### Q9. `d_out > d_in` 在实际中有意义吗？
**有意义**。虽然单次线性投影不增加信息量（信息论），但：
- Transformer 的 FFN 就是 d → 4d → d
- 高维空间中特征更容易**线性可分**
- 多头拼接后总维度往往 ≥ d_in
- 配合多头机制使用收益最大

## 四、Causal Mask（因果掩码）

### Q10. 为什么用 `diagonal=1`？用 `diagonal=0` 会怎样？
- `diagonal=1`：只屏蔽**严格上三角**（未来位置），保留主对角线（自己）→ ✓ 正确
- `diagonal=0`：连主对角线也屏蔽 → 第一行**全是 -inf** → softmax = NaN，训练崩溃

### Q11. 为什么用 `-inf` 而不是 `0` / `-1`？
softmax 内部是 `e^x`：
- `e^0 = 1` → 屏蔽不彻底（仍有权重残留）
- `e^(-inf) = 0` → 权重精确归零，**完美屏蔽**

工程上 fp16 训练时常用 `-1e4`，避免 `inf-inf=NaN` 边界问题。

### Q12. 因果注意力权重矩阵的第一行 `[1, 0, ..., 0]` 代表什么？
第一个 token **100% 只关注自己**——它没有过去，所以输出 z₀ = v₀，没有任何上下文融合。
这反映了自回归模型的根本限制：**序列开头的 token 缺乏上下文**。

## 五、Dropout

### Q13. Dropout 的作用是什么？
**训练时随机丢弃部分神经元/连接**，防止过拟合：
1. 强迫**冗余表示**（不依赖单一神经元）
2. 隐式**模型集成**（每次前向是不同子网络）
3. 打破**共适应**（神经元独立做贡献）

在注意力机制中，Dropout 作用于 softmax 后的 attention weights，**随机切断 token 之间的注意力连接**，防止模型过度依赖特定的"token A 必须看 B"模式。

### Q14. 为什么 Dropout 后未被丢弃的值会放大 1/(1-p) 倍？
为了**保持期望值不变**：
- E[m·x / (1-p)] = (1-p)·x / (1-p) = x
- 这样训练（开 Dropout）和推理（关 Dropout）时下游层接到的信号强度一致
- 这种"训练时放大"的实现叫 **Inverted Dropout**，是 PyTorch 默认做法

### Q15. `model.eval()` 时 Dropout 行为？
**完全失效**，变成恒等函数（输入=输出）。
- 训练：要随机性 → 防过拟合
- 推理：要稳定性 + 可复现 → 关 Dropout
- 数学上"关 Dropout"= 训练时所有随机子网络的期望输出（隐式 ensemble）

实战陷阱：推理前必须 `model.eval()` + `torch.no_grad()` 一起用。

### Q16. `Dropout(p=0.0)` 和不使用 Dropout 有区别吗？
**完全没区别**，PyTorch 内部还有 fast-path 跳过计算。
保留 `Dropout(0)` 层只是为了**配置驱动**——通过改一个数值开关正则化，不用改架构。

## 六、PyTorch 工具

### Q17. `torch.ones(n, n)` 的常见用途？
配合 `torch.tril` / `torch.triu` 构造**因果掩码**：
```python
mask = torch.triu(torch.ones(n, n), diagonal=1)
# 上三角为 1（要屏蔽的未来位置）
```

### Q18. `tensor.unsqueeze(0)` 是什么？
**在第 0 维插入大小为 1 的新维度**，常用于"给单条样本伪造 batch 维度"：
```python
inputs.shape          # (6, 3)
inputs.unsqueeze(0)   # (1, 6, 3) → 可以喂给批处理模型
```
类比：Python 里把单个元素装进 `[x]` 以满足 `list[T]` 类型要求。

### Q19. `nn.Linear` 的权重 shape 和手动 `nn.Parameter` 的区别？
- 手动 `Parameter(torch.rand(d_in, d_out))` → shape `(d_in, d_out)`，配合 `x @ W` 使用
- `nn.Linear(d_in, d_out)` → 内部存储 shape `(d_out, d_in)`（**转置形式**），计算 `x @ W.T`

`nn.Linear` 的存储约定是 `(out, in)` —— 行优先排布更适合硬件加速、初始化更规范。
两者数学等价，但权重互拷需要转置。
